## How to Fill These Guidelines

For each guideline file:

1. Document your project's **actual conventions** (not ideals)
2. Include **code examples** from your codebase
3. List **forbidden patterns** and why
4. Add **common mistakes** your team has made

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.

---

## Pre-Development Checklist

Before changing backend code in this repository:

1. Read [Directory Structure](./directory-structure.md)
2. Read [Database Guidelines](./database-guidelines.md) for any storage or schema work
3. Read [Error Handling](./error-handling.md)
4. Read [Logging Guidelines](./logging-guidelines.md)
5. Read [Quality Guidelines](./quality-guidelines.md)

These documents currently describe a bootstrap baseline. They must be updated as
soon as real backend code establishes stronger conventions.
